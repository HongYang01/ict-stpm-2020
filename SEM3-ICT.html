  • tested by developers (to make sure everything works properly)
Beta Testing

★★★ Without Acceptance Testing

  • Less valuable feedback
  • More error in the system (bug)
  • High expenses to gather data
  • Time consuming to get valuable data
Installation / Conversion
Conversion Explanation
(i)   Direct Conversion - replace old system with new system
(ii)  Parallel Conversion - running old & new system at the same time
(iii) Phased/Gradual Conversion - combining best features of old & new system
(iv) Pilot Conversion - use (i),(ii) or (iii), but only in one pilot site or location

3. Fundamentals of Relational Database System

Data Asset :

  • refer to a system, application, document, database, or web page that companies use to generate revenues

Importance of data asset :

For School :
  • effective in improving student achievement
  • identify the student needs
  • allow school to measure the effectiveness
For Hospital :
  • to improve patient outcomes

Types of data operation :
Stages of data operation / processing :

1. Data capture
2.Validation
  • ensuring data is captured & recorded correctly
3. Sorting
  • arranging data
4. Classifying
  • grouping data
5. Calculation
6. Summarizing
7. Storing
  • hard drive / in cloud
8. Retrieving
9. Reproducing
  • duplicating & store another place
10. Communicating
  • transferring data

Characteristic of good data :

  1. Accurate
  2. Complete
  3. Relevant
  4. Timely (up-to-date)

Data Hierarchy :

  • refer to the systematic organization of data
  • data organized as bit as the lowest to database the highest level
  • more item in data at a higher level

Database :

  • is an organized collection of data
  • independent of application programs

Database System :

  • is a set of procedures and rules applied in order to collect a database effectively, timely and accurately

Database Management System [DBMS] :

  • is a software designed to store, retrieve, define, and manage data in a database
Component Primary Task
  • Data
  • Hardware
  • Software
  • People
  • Procedure
  • Database development
  • Database interrogation

★★★ Important components / features :

DDL (Data Definition Language)
  • define all data that to be stored in database
DD (Data Dictionary)
  • describe logical structure of database
DML (Data Manipulation Language)
  • manipulate information in database

DBMS vs File-based System :

  • Data manipulation
  • Data consistency
  • Data redundancy
  • Data security

Data Integrity

  • is the overall completeness, accuracy and consistency of data

4. Database Systems Planning, Design, and Administration

Data Problems
Data Redundancy
  • is the same piece of data exist in multiple places which causes redundant data
  • will lead to data inconsistency when doing data update
Data Inconsistency
  • same piece of data with different format stored in different table
Data Dependence
  • helps to keep data separated from all programs
  • the immunity of user applications to changes made in the definition & organization of data
Data Abstraction
  • simplify particular body of data representation of the whole
  • make it less complex
Data Anomalies
Insertion
  • is the inability to add data to the database due to absence of other data
Update
  • is a data inconsistency that results from data redundancy and a partial update
Deletion
  • loss of data that is not intended to be removed

Normalization

2NF - must fulfill 1NF & without PD (Partial Dependency)
3NF - must fulfill 2NF & without TD (Transitive Dependency)

★★★ Why use Normalization :

- eliminate data redundancy
- ensuring data dependency
